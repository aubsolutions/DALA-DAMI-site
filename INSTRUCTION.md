# DALA DAMI — инструкция по проекту

## 1. Структура проекта и назначение файлов

- `index.html`  
  Основная страница сайта: header, hero, о компании, каталог, преимущества, как заказать, форма, контакты, footer, модальные окна товара/заказа, панель корзины.

- `style.css`  
  Все стили сайта: цветовая палитра, типографика, адаптивность, карточки товаров, модальные окна, корзина, мобильная кнопка корзины.

- `products.js`  
  Отдельный файл с товарами. Здесь редактируются/добавляются карточки каталога.

- `events.js`  
  Отдельный файл с разделом «Мероприятия». Здесь редактируются карточки событий (дата, место, краткое и полное описание, фото).  
  Раздел на сайте отображается как карусель с кнопками листания и кнопкой «Подробнее».

- `partners.js`  
  Отдельный файл с разделом «Нам доверяют». Здесь редактируются логотипы и названия партнеров.

- `script.js`  
  Вся клиентская логика:
  - рендер каталога из `products.js`
  - рендер мероприятий из `events.js`
  - рендер партнеров из `partners.js`
  - модальное окно «Подробнее»
  - корзина (добавление, удаление, изменение количества)
  - сохранение корзины в `localStorage`
  - расчет итоговой суммы
  - автоподстановка состава корзины в форму заказа
  - отправка формы в `order.php`
  - генерация ссылки заказа в WhatsApp
  - маска/валидация телефона

- `order.php`  
  Обработка формы заказа и отправка письма на email через `mail()`.

- `assets/`  
  Статические файлы:
  - `logo-dala-dami.svg` — логотип (можно заменить на ваш PNG/SVG)
  - `favicon.svg` — иконка сайта
  - `product-1.svg ... product-6.svg` — fallback-изображения товаров (если реальное фото не найдено)
  - `talkan-classic.jpg`, `talkan-hazelnut.jpg`, `talkan-almond.jpg`
  - `granola-nuts.jpg`, `granola-superfood.jpg`, `granola-chocolate.jpg`

## 2. Где менять email для получения заказов

Откройте файл `order.php` и найдите строку:

```php
$recipientEmail = 'dala.dami.2023@gmail.com';
```

Поставьте нужный email (или несколько через запятую):

```php
$recipientEmail = 'your-email@domain.com';
```

В файле уже есть комментарий рядом с этим местом.

## 3. Где менять номер WhatsApp

Откройте `script.js` и найдите:

```js
const WHATSAPP_NUMBER = "77012503020";
```

Замените на ваш номер в международном формате:
- без `+`
- без пробелов
- без скобок

Пример:

```js
const WHATSAPP_NUMBER = "77001234567";
```

Этот номер используется для:
- кнопки WhatsApp в header
- кнопки WhatsApp в контактах
- кнопки «Заказать через WhatsApp» в форме
- кнопки «Заказать через WhatsApp» в модальном окне заказа

## 4. Как редактировать и добавлять карточки товаров

Все товары лежат в `products.js` в массиве `PRODUCTS`.

Сайт мультиязычный: `RU / EN / KZ / 简体中文`.

Пример структуры одного товара (мультиязычно):

```js
{
  id: "talkan-classic-50",
  name: {
    ru: "Талкан Классический",
    en: "Talkan Classic",
    kz: "Классикалық талқан",
    zh: "经典塔尔坎"
  },
  shortDescription: {
    ru: "Короткое описание",
    en: "Short description",
    kz: "Қысқаша сипаттама",
    zh: "简短描述"
  },
  composition: {
    ru: "Состав",
    en: "Ingredients",
    kz: "Құрамы",
    zh: "成分"
  },
  fullDescription: {
    ru: "Полное описание",
    en: "Full description",
    kz: "Толық сипаттама",
    zh: "完整描述"
  },
  volume: {
    ru: "50 г",
    en: "50 g",
    kz: "50 г",
    zh: "50 克"
  },
  price: null,
  image: "assets/talkan-classic.jpg",
  fallbackImage: "assets/product-4.svg"
}
```

Поля:
- `id` — уникальный идентификатор (не повторять)
- `name` — название товара на 4 языках
- `shortDescription` — краткое описание на 4 языках
- `composition` — состав товара на 4 языках
- `fullDescription` — подробное описание на 4 языках
- `volume` — вес/объем на 4 языках
- `price` — число (например `3200`) или `null` для «По запросу»
- `image` — путь к изображению
- `fallbackImage` — запасная картинка, если реальное фото не найдено

### Как изменить существующий товар

1. Откройте `products.js`.
2. Найдите нужный объект в массиве `PRODUCTS`.
3. Измените нужные поля на всех языках (`ru/en/kz/zh`).
4. Сохраните файл и обновите страницу в браузере.

### Как добавить новый товар (новую карточку)

1. Загрузите изображение товара в папку `assets/` (например `assets/product-7.jpg`).
2. В `products.js` добавьте новый объект в массив `PRODUCTS` через запятую.
3. Укажите уникальный `id` (например `chia-bars-300`).
4. Проверьте, что путь к изображению в `image` совпадает с реальным именем файла.
5. Сохраните файл и обновите страницу.

Шаблон для новой карточки:

```js
{
  id: "new-product-id",
  name: { ru: "...", en: "...", kz: "...", zh: "..." },
  shortDescription: { ru: "...", en: "...", kz: "...", zh: "..." },
  composition: { ru: "...", en: "...", kz: "...", zh: "..." },
  fullDescription: { ru: "...", en: "...", kz: "...", zh: "..." },
  volume: { ru: "500 г", en: "500 g", kz: "500 г", zh: "500 克" },
  price: 2500,
  image: "assets/product-7.jpg",
  fallbackImage: "assets/product-1.svg"
}
```

### Важные правила по товарам

- `id` всегда уникальный, иначе корзина и модальное окно будут работать некорректно.
- Если цена нефиксированная, укажите `price: null` и на сайте будет выводиться «По запросу».
- Для мультиязычности обязательны ключи `ru`, `en`, `kz`, `zh` во всех текстовых полях товара.
- Фото можно загружать с фоном: в карточке включен `object-fit: contain`, упаковка не будет обрезана.
- После редактирования `products.js` не нужно менять `index.html` или `script.js` вручную: каталог обновляется автоматически.

## 6. Как работает заказ

### Через email
1. Клиент добавляет товары в корзину.
2. Нажимает «Оформить заказ».
3. Заполняет форму.
4. Состав корзины подставляется автоматически.
5. Форма отправляется в `order.php`.
6. `order.php` отправляет письмо на указанный email.

### Через WhatsApp
1. Клиент нажимает кнопку WhatsApp.
2. Открывается чат с уже сформированным текстом заказа (товары + итог).

## 7. Как загрузить сайт на shared hosting

1. Откройте панель хостинга/FTP.
2. Перейдите в корневую папку сайта (`public_html`, `httpdocs` или аналог).
3. Загрузите все файлы проекта:
   - `index.html`
   - `style.css`
   - `script.js`
   - `products.js`
   - `order.php`
   - папку `assets/`
4. Убедитесь, что на хостинге включен PHP и работает функция `mail()`.
5. Проверьте сайт в браузере:
   - каталог отображается
   - корзина работает
   - заказ через email отправляется
   - WhatsApp-кнопки ведут на нужный номер

---

## 8. Важные заметки

- Проект полностью статический + `order.php`, без Node.js и без WordPress.
- Корзина хранится в `localStorage`, поэтому сохраняется между перезагрузками браузера.
- Если email не отправляется:
  - проверьте правильность `$recipientEmail`
  - проверьте поддержку `mail()` на хостинге
  - при необходимости подключите SMTP через хостинг или PHPMailer (доработка)
